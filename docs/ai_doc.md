# 快捷助手 🚀(Quick Prompts/Agents) 使用说明文档

**文档更新时间：2026年1月28日**

**快捷助手** 是 Anywhere 的核心功能。它不仅仅是简单的“提示词模板”，而是可以被视为一个个独立的微型 AI 应用。您可以为每个助手配置独立的模型、系统指令、触发方式、窗口外观以及关联的工具（MCP/Skill）。

通过合理配置快捷助手，您可以将复杂的 AI 工作流转化为 uTools 中的一个简单指令。

---

## 1. 核心模式说明

在创建助手之前，请先了解两种主要的**显示模式 (Show Mode)**，这将决定助手的交互方式：

### 🚀 快捷输入 (Fast Input)

* **场景**：适用于“阅后即焚”的一次性任务。例如：划词翻译、代码解释、语法纠错。
* **交互**：调用后，会在屏幕中央出现一个轻量级的悬浮条。AI 生成内容后，内容自动复制，您可以直接粘贴/拖拽右侧按钮将内容投放到指定输入框，任务结束后窗口即销毁，不保存对话记录。
* **特点**：极速启动，无干扰，支持流式输出打字机效果。

![快捷输入模式](..\image\快捷输入模式-深色.gif)

### 💬 独立窗口 (Standalone Window)

* **场景**：适用于需要多轮对话、复杂交互的任务。例如：创意写作、代码开发、逻辑推理。
* **交互**：在调用后，会打开一个功能完整的独立聊天窗口（类似 ChatGPT 界面）。
* **特点**：
  * 支持多轮对话上下文。
  * 支持加载 MCP 工具和 Skill 技能。
  * 支持语音输入与输出。
  * 支持文件/图片的多模态交互。
  * 支持自动保存对话记录。

## 2. 创建与配置指南

点击主界面底部的 **“+ 新增快捷助手”** 即可开始配置。

### 2.1 基础设置

* **名称 (Name)**：助手的唯一标识（例如 `Moss`）。这也将是您在 uTools 搜索框中呼出它的关键词。
* **图标 (Icon)**：
  * 支持上传本地图片或粘贴网络图片链接。
  * **内置编辑器**：上传后点击图标可进入编辑器，支持缩放、裁剪和圆角设置。
* **类型 (Type)**：决定了助手如何接收输入。
  * **通用 (General)**：可通过文本指令呼出，也可匹配文本/图片/文件。
  * **文本 (Text)**：仅匹配文本输入（支持正则过滤）。
  * **截图 (Image)**：当您在 uTools 中截图或复制图片时，该助手会出现在选项中。
  * **文件 (Files)**：当您在 uTools 中选中文件时，该助手会出现在选项中。

### 2.2 模型与参数 (Model Parameters)

* **模型选择**：您可以为该助手指定一个专用模型（例如编程助手使用 `Claude-4.5-Sonnet`，闲聊助手使用 `Gemini-3-flash`）。
* **系统提示词 (Prompt)**：这是助手的“灵魂”。定义它的角色、语气、回复格式等（例如：“你是一个资深 Python 工程师，只输出代码，不解释”）。
* **思考预算 (Reasoning Effort)**：针对支持推理的模型，可设置推理强度（默认/低/中/高）。
* **自定义温度**：控制回复的随机性。越低越能保证每次回复的一致性。

### 2.3 窗口与外观 (Window & Appearance)

* **窗口尺寸**：自定义独立窗口的默认宽度和高度。
* **背景设置**：打造个性化界面。
  * **背景图片**：输入图片 URL（支持 http/https 或 file:///）。
  * **不透明度/模糊度**：调整背景的透视效果，支持磨砂玻璃质感。
* **交互行为**：
  * **窗口置顶**：保持窗口在最前方。
  * **失焦关闭**：点击窗口外部时自动关闭（适合查词类助手）。

### 2.4 高级集成 (Integrations)

* **默认 MCP 服务**：选择该助手启动时自动挂载的 MCP 工具（例如：为“联网助手”挂载 `Web Search` MCP）。
* **默认 Skill 技能**：选择该助手启动时自动加载的本地技能库。
* **语音设置**：指定 语音模型（如gpt-4o-mini-audio） 回复语音的音色。

## 3. 标签与管理

当助手数量较多时，使用 **标签 (Tags)** 进行管理至关重要。

* **创建标签**：点击底部的“+ 新增标签”。
* **分配助手**：在标签页下，点击“+”号可将已有助手加入该组。同一个助手可以属于多个标签。
* **批量操作**：
  * **一键启停**：在标签页头部，可一键开启或禁用该标签下的所有助手。
  * **批量替换模型**：点击底部的“替换模型”按钮，可以将所有使用 `GPT-3.5` 的助手一键迁移到 `GPT-4o`，无需逐个修改。

## 4. 常见问题 (FAQ)

**Q1: 为什么我在 uTools 搜索框输入助手名称，却没有出现？**

> **A:** 请检查以下几点：
>
> 1. 助手是否处于“启用”状态。
> 2. 是否设置了特殊的启动方法？例如**文本**、**文件**，此时只有识别到输入文本和文件时才会出现对应助手
> 3. 助手名称是否与 uTools 的其他插件指令冲突？建议使用独特的名称。

**Q2: “文本直接发送”和“补充文本信息”有什么区别？**

> **A:**
>
> * **文本直接发送**：开启后，您在 uTools 输入框中的文字会立即发给 AI 并开始回答。关闭后，文字会填入输入框，等待您二次编辑或添加文件后再发送。
> * **补充文本信息**：部分模型（如 Vertex AI）不允许发送纯空内容。开启此项后，如果您发送纯图片或文件，系统会自动在后台附带一条时间戳文本，以确保存储和请求成功。

**Q3: 正则匹配 (Regex) 是做什么的？**

> **A:** 这属于“文本”类型的高级用法。例如，您可以创建一个“B站视频总结”助手，设置正则为 `/https?:\/\/(www\.)?bilibili\.com\/video\/.*/`。这样，只有当您复制的是 B 站链接时，该助手才会出现在 uTools 的匹配列表中，避免干扰其他操作。

**Q4: 为什么我的背景图片不显示？**

> **A:** Anywhere 会尝试缓存背景图片。请确保图片 URL 是直链（以 .jpg/.png 等结尾），且您的网络环境可以访问该地址。建议使用图床链接或本地 `file:///E:/Images/background.png`。

**Q5: 助手配置丢失了怎么办？**

> **A:** 如果您配置了 WebDAV 备份（在设置页面），可以随时从云端恢复配置文件。建议定期导出配置或启用自动同步。
