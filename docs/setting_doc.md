# 设置 ⚙️(Settings) 使用说明文档

**文档更新时间：2026年1月28日**

「设置」页面是 Anywhere 的控制中心。在这里，您可以对软件的全局行为、界面外观、数据备份及语音功能进行精细化配置。合理的设置能让 Anywhere 更贴合您的使用习惯。

---

## 1. 通用设置 (General Settings)

* **语言 (Language)**：切换软件界面语言。目前支持中文、English、日本語、Русский。
* **主题 (Theme)**：
  * **跟随系统**：自动根据操作系统的深浅色模式切换。
  * **浅色/深色**：强制指定外观。
* **显示成功通知**：开启后，每次 AI 回复完成或操作成功时，右上角会弹出轻提示。
* **固定窗口位置**：开启后，独立对话窗口将记忆上次的位置，而不是每次都在鼠标附近打开。

### 全局行为控制

此处设置将作为所有**新创建**快捷助手的默认值，或一键覆盖所有**已有**助手的设置：

* **全局窗口置顶**：是否让所有 AI 窗口始终显示在最上层。
* **全局失焦关闭**：点击窗口外部时，是否自动关闭 AI 窗口（适合查词、翻译等即用即走场景）。
* **全局自动保存对话**：是否强制所有助手将对话记录保存到本地。

### 输入习惯

* **发送时跳过换行符**：在 uTools 主输入框中粘贴多行文本时，自动合并为一行（适合处理 PDF 复制的断行问题）。
* **Ctrl+Enter 发送**：
  * **开启**：`Enter` 换行，`Ctrl+Enter` 发送。
  * **关闭**：`Enter` 发送，`Shift+Enter` 换行。

## 2. 语音设置 (Voice Settings)

Anywhere 支持 TTS (文本转语音) 输出，让 AI “开口说话”。

* **列表管理**：您可以添加、编辑或删除语音预设。
* **格式**：建议使用 `语音ID` 或 `语音ID - 备注` 的格式。
  * *OpenAI 示例*：`alloy`、`echo`、`shimmer`。
  * *Gemini 示例*：`Puck`、`Charon`、`Kore`（Gemini暂无audio模型，作者的Gemini转OpenAI API的站点支持gemini tts通过对话方式调用）。
* **使用方法**：配置好列表后，您可以在「快捷助手」的设置中，为特定助手绑定一个默认语音。

---

## 3. 数据管理 (Data Management)

数据安全是重中之重。Anywhere 提供了完善的配置导入导出功能。

* **导出配置**：将当前所有的快捷助手、服务商、MCP、Skill(快捷助手默认skill) 配置打包为一个 `.json` 文件。**请妥善保管，切勿泄露（其中包含 API Key）**。
* **导入配置**：从 `.json` 文件恢复所有设置。

## 4. WebDAV 云同步 (Cloud Sync)

这是实现多端同步的核心功能。配置后，您的对话记录和配置文件可以在公司、家庭的多台电脑间无缝漫游。

### 4.1 配置项

* **WebDAV 地址**：您的 WebDAV 服务器地址（例如坚果云、Nextcloud、NAS、Koofr）。
* **用户名/密码**：认证信息。
* **配置路径 (Path)**：存放配置文件的云端目录（默认为 `/anywhere`）。
* **数据路径 (Data Path)**：存放历史对话记录的云端目录（默认为 `/anywhere_data`）。

### 4.2 本地存储

* **本地对话路径**：**必须配置**。请选择一个本地文件夹（如 `D:\Anywhere_Chats`），用于存放自动保存的对话记录。

### 4.3 功能按钮

* **备份到 WebDAV**：手动将当前配置上传备份。
* **管理备份**：查看云端的备份历史，支持一键恢复到任意时间点的版本。

## 5. 常见问题 (FAQ)

**Q1: 为什么修改了“全局窗口置顶”，已有的助手没有变化？**

> **A:** 全局开关在点击切换时，会触发一次批量更新操作，覆盖所有助手。如果您之后单独修改了某个助手的设置，该助手将保持独立设置。再次拨动全局开关可重新统一。

**Q2: WebDAV 连接失败怎么办？**

> **A:**
>
> 1. 检查 URL 是否包含协议头（`http://` 或 `https://`）。
> 2. 部分服务商（如坚果云）需要使用**应用专用密码**，而不是登录密码。
> 3. 确保网络环境可以访问该 WebDAV 服务。

**Q3: 为什么语音朗读没有声音？**

> **A:**
>
> 1. 请确保您使用的 AI 模型支持语音输出功能（目前主要为 OpenAI 的audio 系列）。
> 2. 检查语音 ID 是否拼写正确（区分大小写）。
> 3. 检查系统音量。
