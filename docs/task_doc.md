# 定时任务 ⏰(Scheduled Tasks) 使用说明文档

**文档更新时间：2026年3月8日**

**定时任务** 是 Anywhere 2.0 引入的重磅功能，并在随后的迭代中不断增强。它允许您将 AI 的工作自动化，让 Anywhere 成为您的 24 小时全天候智能员工。

您可以设置任务让 AI 每天早上阅读新闻并生成摘要，或者每小时检查一次服务器状态，甚至让 AI 通过内置的 MCP 工具自我管理和调整任务计划。

---

## 1. 快速开始

在 Anywhere 主界面，点击顶部的 **“定时任务”** 标签页（闹钟图标），即可进入任务管理界面。

![定时任务界面](..\image\定时任务界面.png)

### 1.1 创建任务

1. 点击左侧列表底部的 **“新建定时任务”** 按钮。
2. 输入任务名称（例如：`每日早报`），回车确认。
3. 在右侧详情页配置具体的触发规则和执行内容。
4. 点击右上角的开关 **启用任务**。

### 1.2 核心概念

* **触发规则 (Trigger Rules)**：决定任务何时在后台自动运行。
* **目标助手 (Target Agent)**：决定由哪个 AI 角色来执行任务。
  * *注意*：只有 **“独立窗口”** 模式的快捷助手才能被选中，因为需要生成可视化的输出流，默认使用纯净的**默认助手**。
* **指令内容 (Prompt Content)**：发送给 AI 的具体命令。
  * *建议*：由于任务是“无人值守”执行的，指令必须高度自洽。例如：“请使用 web_search 搜索今日 AI 资讯，总结为 Markdown 格式，并调用 write_file 保存到桌面 news.md 文件中。”

## 2. 触发规则详解

Anywhere 提供了灵活的调度策略，可覆盖绝大多数自动化场景：

* **单次执行 (Once)**：在指定的日期和时间只执行一次，随后自动变为禁用状态。适合临时性的备忘或延迟任务。
* **间隔触发 (Interval)**：每隔 X 分钟执行一次。
  * *每日起始时间*：设置当天的对齐起点（例如 09:00 起每两小时一次）。
  * *生效时间段*：您可以添加一个或多个允许触发的窗口（例如 `09:00 至 12:00`，`14:30 至 17:30`）。任务仅在这些有效时间内触发；若留空，则默认全天 24 小时执行。
  * *场景*：每小时监控服务器内存、白天每30分钟提醒喝水（夜间自动休眠）。
* **每日定时 (Daily)**：每天在固定时间（HH:mm）准时唤醒。
  * *场景*：每日日报生成、自动发送早安邮件。
* **每周定时 (Weekly)**：在每周特定的几天（可多选）的固定时间执行。
  * *场景*：周五下班前写周报、工作日早上查收特定新闻源。
* **每月定时 (Monthly)**：在每月的特定几天（如 1号、15号）的固定时间执行。
  * *场景*：月度账单整理、发薪日提醒。

## 3. 高级配置

### 3.1 全局默认助手 (Default Agent)

如果您不想为仅仅跑一个简单的自动化任务而专门去“快捷助手”里新建一个复杂的角色，可以使用系统内置的 **“默认助手”**。

* 点击左侧底部的 **⚙️ (齿轮图标)**，可以配置该全局默认助手所使用的 AI 模型。
* 默认助手没有任何预设的系统提示词，它是一个白纸状态，完全听从您在任务详情里写的“指令内容”。

### 3.2 临时扩展能力 (Overrides)

您可以为特定任务临时赋予（挂载）额外的能力，而无需去修改目标助手的全局配置。这是实现 AI 操作系统的关键：

* **覆盖 MCP 服务**：任务执行时，允许 AI 调用的工具集合。
  * *默认逻辑*：为了保证自动化任务的执行能力，新建任务时会自动勾选所有的**内置 MCP 服务**（如文件读写、Shell执行、联网搜索等）。您可以根据需要添加第三方的 Github、Notion 等工具。
* **覆盖 Skill 技能**：勾选此任务执行时需要参考的知识库或 SOP 规范。

### 3.3 后台无人值守行为

自动化任务触发时，为了不打断您当前正在进行的工作，Anywhere 做了许多优化：

* **防遮挡弹出**：任务触发时弹出的独立窗口会自动吸附在屏幕右上角（避开主工作区），而非跟随鼠标。
* **自动保存对话 (Auto-save)**：开启后，任务执行产生的所有思考过程和对话结果将自动静默保存到本地。
  * *命名格式*：`[任务名]-[时间戳].json`。
  * *依赖*：必须在全局“设置”中配置好“本地对话路径”。
* **执行完毕自动关闭 (Auto-close)**：开启后，当 AI 完成所有的工具调用和文本输出后，窗口会“阅后即焚”自动销毁，释放内存。

## 4. 历史记录与监控

由于任务是后台自动运行的，您需要一个地方来检查其运行状况。在任务配置页面的最底部，您可以找到 **“最近执行记录”** 面板：

* **状态标签**：直观显示某次执行是成功（绿色）还是异常/超时（红色）。
* **日志追溯**：如果开启了自动保存，每次成功执行会生成一个绿色的文件名（如 `xxx.json`）。**点击该文件名**，Anywhere 会立即在主界面中为您复原那次自动执行的完整上下文和对话内容！
* **立即运行测试 (Play)**：在任务标题右侧，有一个明显的 ▶️ 按钮。配置完毕后，强烈建议点击此按钮立刻触发一次，以检验您的 Prompt 和 MCP 权限是否配置无误。

## 5. 进阶：让 AI 自我管理任务

得益于 Anywhere 强大的 `Super-Agent` 理念和内置的 **Task Manager** MCP，您可以创建一个“秘书”AI，让它来帮您直接管理任务队列，而您只需动动嘴。

**示例对话：**

> **用户**："帮我安排一个任务，从下周一开始，工作日的每天早上9点，自动去科技媒体帮我搜集当天的 AI 前沿资讯，总结成中文并且写到桌面的 news.md 里。"

**AI 的后台操作：**

1. AI 收到指令，自主思考后调用内置工具 `create_task`。
2. AI 自动填充复杂的参数对象：
   * `name`="自动搜集AI资讯"
   * `schedule_type`="weekly"
   * `time_param`="09:00"
   * `weekly_days`=[1,2,3,4,5]
   * `instruction`="调用 web_search 搜索今日 AI 新闻，提取核心信息后调用 write_file 写入到 ~/Desktop/news.md"
3. 任务即刻创建成功并生效，您可以在任务列表中看到它。

## 6. 常见问题 (FAQ)

**Q1: 电脑休眠或 uTools 退出后，任务还会执行吗？**

> **A:** 很遗憾，不会。Anywhere 是依赖于 uTools 底座运行的本地插件。如果电脑休眠或 uTools 进程结束，轮询调度器将暂停。唤醒后，如果错过了触发时间点（例如设定的 12:00，但在 12:05 唤醒），系统会跳过该次执行，等待下一个周期。

**Q2: 为什么我的任务触发了，但是窗口弹出来一片空白或者一直转圈？**

> **A:**
>
> 1. 请检查您的网络连接。
> 2. 请确认您在【全局默认助手】或指定的【快捷助手】中配置了有效的模型和足额的 API Key。
> 3. 检查指令是否过于复杂导致 AI 卡死，建议在任务名旁边点击 ▶️ 进行一次手动测试调试。

**Q3: 多个任务同一时间触发会发生什么？**

> **A:** Anywhere 的架构支持多窗口并发。它们会同时弹出各自的独立窗口并独立向 API 发送请求。不过受限于系统内存和网络带宽，建议尽量错开高频重负载任务的执行时间。

**Q4: 为什么我清空了历史记录，但本地的 json 文件还在？**

> **A:** 点击卡片上的“清空”按钮时，系统会弹窗警告。通常情况下，Anywhere 会尝试一并删除本地挂载的日志文件。如果文件正在被占用（如会话仍在进行，过程中会保存历史记录），可能导致清理失败，此时需要您前往本地文件夹手动删除。
